## Communication Plan

The group will serve as the workforce using the WhatsApp, Zoom, tutorials discussions and the wonders of GitHub. Quick updates, sharing the screenshots and configuration issues will be addressed with WhatsApp. OpenWRT settings will be reviewed using Zoom, as well as diagrams, and Wireshark captures, Markdown files, and the final video. Contact between the group will be at least twice a week. One will be during in or post tutorials time and the other will be online provided it is appropriate in the tasks or writing sections of work to be checked. The members will frequently utilize GitHub to make sure that packet captures, reports, diagrams, screenshots and files are frequently updated.

## Task Allocation

The project activities were shared among the group members to make sure that all the technical, documentation and security requirements were met effectively. The following shared responsibilities were involvement in project planning, business scenario development, production network design, risk assessment, security control recommendations, project reflection, as well as preparation of the final demonstration video. Assignments were done based on individual areas of technical specialisation to enable every member work in their areas of specialisation whilst collaborating with others on the project.

### Task Distribution:

- **Network Diagram (Syed Omair)**
  - Create and document the lab network topology.
  - Export diagrams and verify IP addressing.
  - Evidence: Draw.io files and exported diagrams.

- **Website Setup (Syed Omair)**
  - Configure and test the OpenWRT-hosted website.
  - Verify browser access and web server functionality.
  - Evidence: Browser screenshots.

- **Firewall Rules (Syed Omair)**
  - Test HTTP, SSH, ICMP, and management access rules.
  - Validate firewall filtering behaviour.
  - Evidence: Firewall and testing screenshots.

- **Production Network Design (All Students)**
  - Design the production network architecture.
  - Develop IP addressing scheme and topology documentation.
  - Evidence: Network diagrams and IP allocation tables.

- **System Hardening (Sania Fatima)**
  - Change default credentials.
  - Configure SSH key authentication.
  - Disable unnecessary services and improve system security.
  - Evidence: Hardening screenshots and explanations.

- **Traffic Analysis (Sania Fatima)**
  - Capture and analyse HTTP and SSH traffic.
  - Review packet captures using Wireshark.
  - Evidence: PCAP files and screenshots.

- **Risk Assessment (All Students)**
  - Conduct TVAMatrix security risk assessment.
  - Evaluate threats, vulnerabilities, and impacts.
  - Evidence: Risk assessment spreadsheet and screenshots.

- **Security Controls (All Students)**
  - Recommend and document security controls.
  - Justify controls based on identified risks.
  - Evidence: security.md

- **Reflection and Documentation (All Students)**
  - Document project contributions, challenges, and lessons learned.
  - Maintain GitHub commit history and project records.
  - Evidence: reflection.md and GitHub screenshots.

- **Demonstration Video (All Students)**
  - Record system demonstration and testing outcomes.
  - Present firewall rules, website access, and packet captures.
  - Evidence: Submitted MP4 video.

## GitHub Contribution Plan

The students will add their own work to the GitHub repository. The commits will occur when there is any significant development - addition of Markdown material, Screen shots, Diagrams, Packet Capture files, Risk Assessment updates, Report Updates etc. This will prove a record of individual input and keep a proper track of progress. Students will not have to pour it all at the end of the project, they will do it regularly.
